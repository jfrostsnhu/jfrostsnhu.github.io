The artifact selected for enhancement is the CS-360 Mobile Programming: Inventory Manager Application, originally developed between April and June 2024 as part of the Mobile Programming course. This Java-based Android application was designed to help warehouse associates track inventory, providing a simple and effective way to manage stock levels, receive low-stock alerts, and enforce role-based authorization. I chose this artifact for my ePortfolio because it demonstrates my ability to design, develop, and enhance software solutions. While the original project successfully met its objectives, I wanted to evolve it into a full-stack web application, improving its scalability, security, and accessibility.

This enhancement involved converting the Android-based Inventory Manager into a full-stack .NET 8 Blazor Server application transforming it from a platform-dependent mobile app to a scalable, web-accessible solution. The front-end was migrated from Android XML layouts to Blazor components, allowing users to access the system from any device with a web browser. Additionally, Identity Framework was integrated to introduce secure role-based authentication and authorization ensuring that user permissions are managed more effectively. This enhancement greatly improved the usability, scalability, and security of the application. By migrating to Blazor Server, the system is now platform independent, allowing employees to manage inventory from any device with internet access. Security was also a major focus, with Identity Framework ensuring encrypted authentication, input validation preventing invalid or malicious data entry, and structured error handling improving system stability.

From a business perspective, this enhancement transforms the Inventory Manager from a basic mobile tool into a scalable enterprise solution. Businesses can now leverage real-time inventory updates, secure authentication, and cloud-ready infrastructure while ensuring that inventory operations run more efficiently and securely. Employees can update stock levels from any location, reducing errors and improving workflow efficiency. The improved security model ensures that only authorized users can manage inventory, reducing the risk of unauthorized modifications.

The process of enhancing this artifact was a significant learning experience. Transitioning from .NET Core 3 to .NET 8 required adapting to the removal of namespaces and top-level statements reinforcing my ability to learn and adapt to new technologies. As a first time Blazor developer, I faced challenges in structuring components, customizing UI elements, and overriding Bootstrap styles. Overcoming these challenges strengthened my problem-solving skills, technical expertise, and ability to integrate complex solutions into a functional and scalable application.
