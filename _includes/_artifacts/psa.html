The CS-499 Capstone Project serves as the culmination of my academic journey showcasing my ability to design, develop, and refine a fully functional software solution aligned with industry best practices. The original artifact selected for enhancement is the Inventory Management Android application which I developed during CS-360 Mobile Architecture and Programming between April and June 2024. The original purpose of this mobile application was to allow warehouse associates to manage inventory, providing low-stock alerts, role-based authentication, and CRUD operations for inventory tracking. While the project met its initial objectives, it was designed as a standalone Android application, limiting its scalability, accessibility, and long-term viability in an enterprise setting.

To ensure a fully modernized solution, I used this artifact for all three enhancements, Software Design & Engineering, Algorithms & Data Structures, and Databases, each addressing key areas of improvement. The first enhancement involved converting the application from an Android mobile app to a full-stack web-based solution using .NET 8 Blazor Server, making it accessible across multiple devices while improving security and maintainability. The second enhancement optimized low-stock notifications by replacing the inefficient linear search algorithm with an AVL Tree to ensure faster and more scalable inventory alerting. Finally, the third enhancement migrated the database from SQLite to Microsoft SQL Server to improve data integrity, security, and multi-user support. Together, these enhancements transform the project into an enterprise-ready inventory management system, improving scalability, security, and accessibility.

Prior to implementing enhancements, I conducted a comprehensive code review of the original artifact, identifying areas for improvement in functionality, security, efficiency, and maintainability. This analysis provided the foundation for defining how to address the enhancements while ensuring they aligned with industry standards and best practices.
