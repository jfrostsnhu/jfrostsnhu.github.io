For this enhancement, I'm using the same artifact which is the Inventory Manager Android application developed in CS-360 Mobile Programming between April and June 2024. The initial implementation utilized a local SQLite database which was effective for lightweight applications but lacked scalability, availability, and multi-user access capabilities. While SQLite functioned well for a standalone mobile application it did not fit the bill of an enterprise-ready solution. As the project evolved into a web-based application, a more robust and scalable database infrastructure was necessary to ensure data integrity, concurrency control, and secure multi-user access.

To align the project with enterprise-level standards I migrated the database from SQLite to Microsoft SQL Server, a powerful, scalable, and secure relational database management system. This enhancement improved performance, security, and data availability, enabling the application to support multiple users across web and mobile platforms. Additionally, I integrated Entity Framework as an Object-Relational Mapping tool to ensure efficient, maintainable, and secure database operations. The new database structure was designed with normalized tables, incorporating primary and foreign keys to enforce referential integrity and reduce redundancy. This change ensures data consistency while allowing for scalable interactions between multiple services. Furthermore, security best practices were implemented, including parameterized queries to prevent SQL injection attacks, ensuring that user input is securely processed.

Migrating to Microsoft SQL Server enhances the system's security, scalability, and reliability, making it a more viable solution for businesses managing larger inventories. The ability to handle multiple concurrent users ensures that inventory updates can be performed simultaneously across different devices, improving operational efficiency. With remote access capabilities, businesses can securely manage inventory from any location, supporting modern distributed work environments. Additionally, integrating Entity Framework streamlines database interactions, reduces development complexity and ensures future enhancements easier to implement and maintain.

This enhancement required a deep understanding of database migration processes and the differences between SQLite and SQL Server. The transition involved modifying SQL queries, designing normalized relational tables, and adapting the data access layer to work seamlessly with Entity Framework and LINQ queries. Additionally, ensuring data integrity during migration was an essential aspect, requiring thorough testing to confirm that all records were transferred accurately without loss or corruption.
